# CMU Resume Guide

CMU-style guidance is especially useful for technical and business-technology resumes where hard skills and project evidence matter.

## What this node is best for

- Source type: university
- Target role: general
- Target country: USA
- Resume length: Usually one page for students and early-career recruiting; academic CVs may be longer.

## Preferred section order

1. Contact
2. Education
3. Technical Skills
4. Experience
5. Projects
6. Leadership
7. Activities

## Formatting rules

- fontRecommendation: Readable serif or sans-serif resume font; keep one family and two weights at most.
- margin: 0.5-0.75 inch for one-page resumes; 0.75-1 inch for academic CVs.
- bulletStyle: Single-level bullets, 1-2 lines each, parallel punctuation.
- dateFormat: Consistent month/year or year-only format, right-aligned when layout supports it.
- useColumns: false
- atsFriendly: true

## Writing rules

- Tone: Professional, concise, evidence-first, and school-career-center aligned.
- Bullet formula: Action verb + what was done + scope/context + result/impact
- Quantification: Prefer impact metrics, team/project scope, audience size, budget, ranking, or time saved. Attach a number, scale, frequency, audience, runtime, revenue, cost, accuracy, conversion, latency, or time-to-completion metric whenever the source material supports it. When exact metrics are absent, use bounded scope statements such as team size, dataset size, user group, deliverable count, or project duration. Never invent metrics. Mark missing metrics as an improvement question instead of fabricating proof.
- Avoid: objective statement without value, personal pronouns, unexplained acronyms, responsible for, helped with, worked on, participated in, various, many, good, great, hardworking

## Reusable prompt snippet

- When applying CMU Resume Guide rules, preserve conventional section names, prioritize the most relevant evidence first, and do not add unsupported content.
- Rewrite bullets using Action verb + what was done + scope/context + result/impact; each bullet should start with a strong verb, name the method or tool, include scope, and end with an outcome when evidence exists.
