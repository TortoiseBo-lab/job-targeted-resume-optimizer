# Abstracted learning profile: BCG

BCG rules abstract recurring job-description signals into resume language. Target signals: strategic insight, growth, transformation, analytics, client partnership, innovation.

## What this node is best for

- Source type: company
- Target role: Consulting
- Target country: general
- Resume length: One page for internship/new-grad recruiting; concise two-page resumes only for senior roles.

## Preferred section order

1. Contact
2. Education
3. Experience
4. Leadership
5. Projects
6. Skills
7. Interests

## Formatting rules

- fontRecommendation: Readable serif or sans-serif resume font; keep one family and two weights at most.
- margin: 0.5-0.75 inch for one-page resumes; 0.75-1 inch for academic CVs.
- bulletStyle: Single-level bullets, 1-2 lines each, parallel punctuation.
- dateFormat: Consistent month/year or year-only format, right-aligned when layout supports it.
- useColumns: false
- atsFriendly: true

## Writing rules

- Tone: Strategic, analytical, collaborative, and transformation-focused.
- Bullet formula: Action + structured analysis + client/business question + recommendation/result
- Quantification: Map evidence to company signals: strategic insight, growth, transformation, analytics, client partnership, innovation. Attach a number, scale, frequency, audience, runtime, revenue, cost, accuracy, conversion, latency, or time-to-completion metric whenever the source material supports it. When exact metrics are absent, use bounded scope statements such as team size, dataset size, user group, deliverable count, or project duration. Never invent metrics. Mark missing metrics as an improvement question instead of fabricating proof.
- Avoid: company-name stuffing, mission parroting without evidence, unsupported culture-fit claims, responsible for, helped with, worked on, participated in, various, many, good, great, hardworking

## Reusable prompt snippet

- When applying BCG rules, preserve conventional section names, prioritize the most relevant evidence first, and do not add unsupported content.
- Rewrite bullets using Action + structured analysis + client/business question + recommendation/result; each bullet should start with a strong verb, name the method or tool, include scope, and end with an outcome when evidence exists.
