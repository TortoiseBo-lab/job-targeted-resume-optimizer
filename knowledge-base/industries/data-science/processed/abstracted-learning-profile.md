# Abstracted learning profile: Data Science

Data Science rules convert role expectations into sections, keywords, and bullet formulas. Key language signals: machine learning, experimentation, feature engineering, Python, SQL, model evaluation, causal inference, MLOps.

## What this node is best for

- Source type: industry
- Target role: Data Scientist
- Target country: general
- Resume length: One page preferred for early-career roles; two pages only for substantial relevant experience.

## Preferred section order

1. Contact
2. Education
3. Skills
4. Experience
5. Projects
6. Research
7. Publications

## Formatting rules

- fontRecommendation: Readable serif or sans-serif resume font; keep one family and two weights at most.
- margin: 0.5-0.75 inch for one-page resumes; 0.75-1 inch for academic CVs.
- bulletStyle: Single-level bullets, 1-2 lines each, parallel punctuation.
- dateFormat: Consistent month/year or year-only format, right-aligned when layout supports it.
- useColumns: false
- atsFriendly: true

## Writing rules

- Tone: Clear, technical, and outcome-oriented.
- Bullet formula: Action + dataset/model/method + evaluation metric + decision or product impact
- Quantification: Prioritize machine learning, experimentation, feature engineering, Python, SQL evidence where supported by the user's experience. Attach a number, scale, frequency, audience, runtime, revenue, cost, accuracy, conversion, latency, or time-to-completion metric whenever the source material supports it. When exact metrics are absent, use bounded scope statements such as team size, dataset size, user group, deliverable count, or project duration. Never invent metrics. Mark missing metrics as an improvement question instead of fabricating proof.
- Avoid: generic industry buzzwords without proof, unsupported expertise claims, responsible for, helped with, worked on, participated in, various, many, good, great, hardworking, detail-oriented

## Reusable prompt snippet

- When applying Data Science rules, preserve conventional section names, prioritize the most relevant evidence first, and do not add unsupported content.
- Rewrite bullets using Action + dataset/model/method + evaluation metric + decision or product impact; each bullet should start with a strong verb, name the method or tool, include scope, and end with an outcome when evidence exists.
