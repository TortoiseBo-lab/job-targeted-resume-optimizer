# ModernCV

moderncv pattern works best for formal CVs with a longer evidence record and traditional section taxonomy.

## What this node is best for

- Source type: style
- Target role: general
- Target country: general
- Resume length: Flexible CV length.

## Preferred section order

1. Contact
2. Education
3. Experience
4. Publications
5. Projects
6. Skills
7. Languages
8. References

## Formatting rules

- fontRecommendation: Readable serif or sans-serif resume font; keep one family and two weights at most.
- margin: 0.5-0.75 inch for one-page resumes; 0.75-1 inch for academic CVs.
- bulletStyle: Classic CV structure with clean typography; suited to academic/professional CVs.
- dateFormat: Consistent month/year or year-only format, right-aligned when layout supports it.
- useColumns: false
- atsFriendly: false

## Writing rules

- Tone: Scholarly, precise, and complete.
- Bullet formula: Action + method + corpus/sample + output
- Quantification: Adjust bullet density to style: ATS and banking can be denser; minimal and modern should be more selective. Attach a number, scale, frequency, audience, runtime, revenue, cost, accuracy, conversion, latency, or time-to-completion metric whenever the source material supports it. When exact metrics are absent, use bounded scope statements such as team size, dataset size, user group, deliverable count, or project duration. Never invent metrics. Mark missing metrics as an improvement question instead of fabricating proof.
- Avoid: decorative formatting that weakens parser readability, responsible for, helped with, worked on, participated in, various, many, good, great, hardworking, detail-oriented, fast learner

## Reusable prompt snippet

- When applying ModernCV rules, preserve conventional section names, prioritize the most relevant evidence first, and do not add unsupported content.
- Rewrite bullets using Action + method + corpus/sample + output; each bullet should start with a strong verb, name the method or tool, include scope, and end with an outcome when evidence exists.
