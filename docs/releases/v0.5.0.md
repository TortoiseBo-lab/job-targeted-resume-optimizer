# v0.5.0 - Stronger Word/PDF Layout Engine

This release upgrades document generation from simple export to resume-aware layout.

## What's New

- Added automatic `standard`, `compact`, and `dense` layout profile selection based on estimated resume length.
- Improved Word output with section divider lines, widow control, heading keep rules, Word settings, and consistent resume spacing.
- PDF export now tries LibreOffice conversion first when a working `soffice` or `libreoffice` command is available.
- Auto PDF mode falls back to the internal renderer if LibreOffice is missing or broken.
- Internal PDF wrapping now uses page-width estimation instead of fixed character counts.
- Export output now reports the selected layout profile and PDF engine.

## Why This Matters

The final resume should be a submit-ready document, not a Markdown-looking draft. This version makes the Word file the primary layout source and gives PDF generation a stronger path when a local converter is available.

## Command

```bash
npm run export -- --input outputs/<company-role>-targeted-resume.md
```

Optional:

```bash
npm run export -- --input outputs/<company-role>-targeted-resume.md --pdf-engine libreoffice
npm run export -- --input outputs/<company-role>-targeted-resume.md --pdf-engine internal
```
