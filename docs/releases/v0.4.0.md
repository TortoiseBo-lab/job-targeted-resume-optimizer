# v0.4.0 - Final Resume Documents First

This release changes the skill behavior from "Markdown package first" to final document delivery.

## What's New

- Word and PDF are now the main user-facing deliverables.
- Markdown is treated as an internal export source unless the user explicitly asks for it.
- Optimization notes, evidence gaps, follow-up questions, and missing-detail markers must stay out of the resume.
- The skill now creates a separate optimization report for analysis and next questions.
- The exporter removes legacy `[Need detail: ...]` markers and report sections from resume exports by default.
- Word and PDF exports now include section dividers for resume sections.
- The default exporter formats are now `docx,pdf`; HTML is still available only when requested with `--formats html`.

## Expected Output

```txt
outputs/<company-role>-targeted-resume.docx
outputs/<company-role>-targeted-resume.pdf
outputs/<company-role>-optimization-report.md
```

## Why This Matters

The resume document should be an application-ready final artifact. Strategy notes and missing facts are useful, but they belong in a separate report, not in the resume a candidate would submit.
