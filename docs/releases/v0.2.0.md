# v0.2.0 - Word/PDF Export And Usage Clarity

This release turns the package into a more usable resume production workflow.

## What's New

- Added a no-dependency export script:

  ```bash
  npm run export -- --input outputs/my-targeted-resume.md
  ```

- The exporter creates:
  - `.docx` Word file.
  - `.pdf` file.
  - `.html` preview file.

- Added a runnable example:
  - `examples/inputs/sample-resume.md`
  - `examples/inputs/google-swe-intern-jd.md`
  - `examples/outputs/example-targeted-resume.md`
  - `examples/outputs/example-targeted-resume.docx`
  - `examples/outputs/example-targeted-resume.pdf`
  - `examples/outputs/example-targeted-resume.html`

- Updated the Claude Code skill so that when users ask for Word or PDF, it generates Markdown first and then runs the export script.

- Added a public usage guide:
  - `docs/USAGE.md`

## Usage

Try the example:

```bash
npm run export:example
```

Export your own generated resume:

```bash
npm run export -- --input outputs/google-swe-intern-targeted-resume.md
```

## Why This Matters

The project now has a clearer end-to-end shape:

1. Use the prompt or Claude Code skill to generate a targeted resume in Markdown.
2. Review missing evidence marked by the AI.
3. Edit the Markdown until the facts are correct.
4. Export the final version to Word and PDF.

## Known Limitations

- The built-in PDF exporter is intentionally lightweight and best for English resume drafts.
- The Word exporter creates a clean editable `.docx`, but it is not yet a full design-template engine.
- Future releases should add richer layout templates and stronger visual QA.
